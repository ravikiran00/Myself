GCO Credit Card Receipt Portal

Duration : 1 Year

Role : .Net Full Stack Developer

Project description :

I was Team Lead for this Project. GCO Credit card portal project was developed for internal use only. GCO has different vendors and projects on which GCO financial, or Higher team do transactions against projects. Around 15 or 16 people there which they are doing the transaction on behalf of GCO. So whatever transaction they are doing they carry the payment slip against the transaction. GCO finance team received transaction statement from bank, and they collect payment slip against that, they are doing without any system and maintaining this information was so difficult. So, to overcome this documentation headache we have developed this application.

This application has two parts one is Admin Module and Second one is User Module. Admin module has a right to upload the statement against the user. If admin uploads the bank statement against any user, then that user will receive the mail that you’re this month statement uploaded please upload the receipts against that. So, users upload the payment slip against every debited transaction. Once he uploads the slip then all admins will receive mail that this-this user has uploaded payment slip against this transaction id please take action against that. So, admin will approve or reject transaction based on his knowledge and whatever action it taken that will reflect on User module to that user. So, using this application we will maintain the data month wise.

We have Used Azure Storage Explorer for Storing Bank statement files as well as receipts for those users. This application has oneSignal Browser notification functionality, if user subscribe this functionality, then he will receive browser notification on desktop. We have added a camera service for users as well to upload receipt by clicking with mobile camera and will store it into his gallery. After he uploads the receipt in gallery, he will use afterwords for any transaction.
